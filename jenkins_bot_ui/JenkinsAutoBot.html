Autobots

Smart interactions with Jenkins

About

Autobot is an automation bot that provides a rich and secure access to Jenkins software. This infrastructure allows developer and integrators perform Jenkins operations without the need for multiple clicks on the Jenkins website.

Some Jenkins' jobs takes a lot of time to run hence user has to wait for long time and check their status time to time. Meanwhile if a user managing multiple Jenkins jobs then in order to check status of each job, he needs to navigate multiple pages with multiple clicks. This process is time consuming. Our solution tries to solve this problem by providing a chat/Web based single screen dashboard with query functionality. User can also trigger a regression suite for build validation.

Features

  • Retrieve build status of Jenkins job(s).
  • Retrieval of failure/success details of a Jenkins job.
  • Retrieval of failure/success details of a Jenkins job.
  • Get the details of the current running job.
  • Disable/enable a job(s).
  • Stop a Jenkins job(s).
  • Automate the invocation of regression suite during deployment.
  • Analytics for a Jenkins job based on its attributes.

How to use

Autobot site has an embedded chat application which is easy to use. Prerequisites are to have a Jenkins account. The chat application will be the means by which the user can trigger jobs and get information required.

Users can choose from the set of jobs provied under Chat Integration menu. The environment for running the job will also be required by the application to narrow down the set and give a more accurate result.

Information

  • QA
